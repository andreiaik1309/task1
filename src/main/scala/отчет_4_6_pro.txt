1. Сортировка вставками (Insertion Sort)
Описание:
Алгоритм проходит по массиву и на каждом шаге вставляет текущий элемент на правильное место в уже отсортированной части массива.
Big O:
Средний и худший случаи: O(n^2)
Лучший случай (уже отсортированный массив): O(n)

Результаты замеров времени выполнения:
Для массива размером 100: 8,908,208 наносекунд
Для массива размером 1000: 5,147,833 наносекунд
Для массива размером 10000: 116,140,708 наносекунд
2. Сортировка пузырьком (Bubble Sort)
Описание:
Алгоритм проходит по массиву несколько раз, сравнивая и меняя местами соседние элементы, пока весь массив не будет отсортирован.
Big O:
Средний и худший случаи: O(n^2)
Лучший случай (уже отсортированный массив): O(n)

Результаты замеров времени выполнения:
Для массива размером 100: 2,477,667 наносекунд
Для массива размером 1000: 25,407,000 наносекунд
Для массива размером 10000: 872,352,209 наносекунд
3. Сортировка слиянием (Merge Sort)
Описание:
Алгоритм разделяет массив на две части, рекурсивно сортирует их, а затем объединяет в один отсортированный массив.
Big O:
Средний, худший и лучший случаи: O(n log n)

Результаты замеров времени выполнения:
Для массива размером 100: 1,349,042 наносекунд
Для массива размером 1000: 6,456,250 наносекунд
Для массива размером 10000: 58,274,125 наносекунд
4. Быстрая сортировка (Quick Sort)
Описание:
Алгоритм выбирает опорный элемент, разделяет массив на элементы, меньше и больше опорного, и рекурсивно сортирует обе части.
Big O:
Средний случай: O(n log n)
Лучший случай (оптимальный выбор опорного элемента): O(n log n)
Худший случай (при неудачном выборе опорного элемента): O(n^2)

Результаты замеров времени выполнения:
Для массива размером 100: 9,416,916 наносекунд
Для массива размером 1000: 8,767,917 наносекунд
Для массива размером 10000: 30,047,333 наносекунд
Выводы:
Сортировка слиянием и сортировка вставками оказались эффективными для небольших размеров массива.
Быстрая сортировка показала хорошую производительность для средних размеров массивов.
Сортировка пузырьком оказалась неэффективной для больших массивов.
В целом, выбор алгоритма сортировки зависит от размера массива и требований к производительности.
